## 2. Nouvelles Règles de Développement Impératives

### RÈGLE 1 : Interdiction de l'Invention Créative non sollicitée

L'IA **ne doit jamais** modifier les styles visuels, les couleurs ou le thème général d'un projet existant. Elle doit s'adapter strictement à la charte graphique présente dans les fichiers CSS originaux.

### RÈGLE 2 : Respect Absolu du Code Source

Le code fourni par l'utilisateur (via archives ou templates) est la **seule vérité**. L'IA a interdiction de "l'améliorer" ou de le "moderniser". Elle doit se limiter à réparer les bugs techniques spécifiques demandés.

### RÈGLE 3 : Stabilité des Mise en Page (Layout)

Toute directive concernant l'organisation spatiale (ex: "une seule colonne") doit être appliquée rigoureusement sans tentative de passage en grille ou multi-colonnes.

### RÈGLE 4 : Priorité à la Continuité

Avant toute modification, l'IA doit vérifier les fichiers de référence (dossiers de sauvegarde comme `batimentRenovation`) pour garantir que le nouveau code s'inscrit dans la continuité exacte du travail de l'utilisateur.
